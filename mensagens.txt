    Header: (todas as mensagens vão ter o header!)
        -> ID Pedido
        -> Tipo de Mensagem?
        -> ID nodo de origem (IP, nome)
        -> ID nodo antecessor

    Ack:
        -> ID do Pedido a que corresponde a confirmação

Construção da rede

    Multi-Cast

        Ping:
            -> Lista de vizinhos N1 (nível 1)
            -> Lista de vizinhos N2 (nível 2)

    Uni-Cast

        Pong:
            -> Lista de vizinhos N1
            -> lista de vizinhos N2
            -> ID do Ping a que está a responder

        nbrConfirmation:
            -> Lista de informações sobre os ficheiros que possui
        addNbrs:
            Lista de vizinhos

        Quit: (só tem mesmo o header)

Manutenção vizinhos

    Alive:
        -> Lista de vizinhos N1
        -> Lista de vizinhos N2

Ficheiros

    UpdateTable:
        -> Lista de ficheiros para adicionar (pode ir vazia)
        -> Lista de ficheiros para remover (pode ir vazia)
        Nota: Nunca poderão ir as duas vazias, se assim for ignoramos

    ContentDiscovery:
        -> Nome do ficheiros
    
    ContentOwner:
        -> Info do ficheiro (nome, tamanho, checksum ...)
